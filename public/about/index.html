About Dhruv: A Journey into C Networking

Dhruv (named after the North Star, symbolizing guidance and a fundamental point of reference) is a personal project aimed at demystifying the intricate world of low-level networking and the HTTP protocol. Built entirely from scratch in **C for Linux**, this server eschews frameworks and libraries to provide a raw, unadulterated understanding of how web servers truly function.

The Motivation

In an era dominated by high-level abstractions and powerful frameworks, it's easy to lose sight of the foundational technologies that power the internet. Dhruv was born from a desire to strip away these layers and confront the core mechanics:

  • Mastering Sockets: To gain hands-on experience with the POSIX socket API, including `socket()`, `bind()`, `listen()`, `accept()`, `send()`, and `recv()`.
  • Understanding HTTP/1.1: To manually parse HTTP request lines and headers, and construct valid HTTP responses, deepening the understanding of the protocol's stateless nature.
  • Low-Level C Programming: To reinforce C programming skills, focusing on memory management, string manipulation, error handling, and system calls in a real-world application.

What Dhruv Does (Currently)

At its current stage, Dhruv is a barebones server capable of:

  • Serving static files (like this `index.html`, CSS, and JavaScript) from its `public/` directory.
  • Handling basic HTTP GET requests.
  • Managing multiple client connections (though currently in a simple, blocking manner which is part of the learning curve).

Future Aspirations (and current limitations)

This project is a continuous learning experience. Future goals include:

  • Implementing non-blocking I/O using `epoll` for better concurrency.
  • Adding robust error handling and logging.
  • Support for more HTTP methods (POST, HEAD).
  • Basic routing capabilities.
  • Exploring more advanced security considerations (though always prioritizing educational clarity).

Important: Dhruv is an **educational project** and is **not secure for production use**. It is likely to contain vulnerabilities and is intended solely for learning and experimentation in controlled environments.

Thank you for exploring Dhruv. It's a testament to the power of foundational computing and the joy of building from scratch.
